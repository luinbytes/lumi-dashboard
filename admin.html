OpenClaw Admin - How to Use
Dashboard Overview
What is this dashboard?
The OpenClaw Admin Dashboard is a web-based interface for managing your OpenClaw agents, editing their configuration files, and monitoring their status. It provides direct access to agent identity files (SOUL.md), memory (MEMORY.md), and other critical configuration.
Real-time status
The dashboard shows real-time connection status, the number of active agents, and environment context (local/production). Status indicators in the sidebar show which agents are currently online and responding.
Integration: Status is fetched from the /api/status endpoint every 10 seconds. Agent online/offline state is determined by OpenClaw's internal health checks.
Agent Management
Selecting an Agent
Click on any agent in the sidebar to select it. The selected agent's files will be loaded in the main editor area. The active agent is highlighted with a darker background.

Restarting an Agent
Click the "Restart" button in the header to restart the currently selected agent. A confirmation dialog will appear before the restart is initiated. This sends a restart command to the OpenClaw gateway.
Integration: Restart sends a POST request to /api/restart with the agent ID. The OpenClaw gateway handles the actual restart process.
Agent Skills Count
Each agent card shows the number of skills available to that agent. Skills are the tools and capabilities the agent can access (e.g., web search, file editing, GitHub API).
File Editing
File Tabs
The top tab bar shows available files for the selected agent. Common files include:
SOUL.md - Agent identity, personality, and behavioral rules
MEMORY.md - Long-term memory, lessons learned, important facts
USER.md - Information about the user being served
TOOLS.md - Local tool configurations and notes
Line Numbers
The editor shows line numbers on the left side for easy reference. These update automatically as you type and scroll.
Saving Changes
Click the "Save" button in the right action bar (or press Ctrl+S) to save your changes. A confirmation message will appear at the bottom right. Changes are immediately written to the agent's workspace files.
Integration: Save sends the file content to /api/file endpoint, which writes directly to the workspace. No server restart required - changes take effect immediately.
Undo/Redo
Use the undo/redo buttons in the action bar to navigate your edit history. Currently, this tracks changes within the current editing session.
Format Markdown
Click the "Format" button to apply basic markdown formatting (headings, bold, code blocks). This is a lightweight formatter - for complex formatting, use a dedicated markdown editor.
Navigation Sections
Skills
View and manage agent skills. Shows all available skills and their configurations. Skills define what tools and capabilities agents can access.
Channels
Manage messaging channels (Discord, Telegram, WhatsApp, etc.). View active channels, their status, and configuration.
Cron Jobs
Schedule automated tasks and reminders. Create one-shot or recurring jobs (e.g., daily summaries, hourly checks, weekly reports).
Integration: Cron jobs are managed by the OpenClaw gateway and trigger system events or agent turns at specified times.
Config / Settings
Global configuration for OpenClaw and the dashboard. Includes theme settings, API keys, and behavior preferences.
Behavior Flow
1
Dashboard loads
Fetches initial data from /api/dashboard endpoint (agents list, port, environment)
2
User selects agent
Clicking an agent in sidebar triggers selectAgent(), loads agent's files via /api/file?agent=X&file=Y
3
User edits file
Changes are made in the textarea. Line numbers update dynamically. No validation yet.
4
User saves file
Save button sends POST to /api/file with agent, file, content. File is written to workspace. "All changes saved" confirmation appears.
5
Agent behavior updates
Next heartbeat or message processing cycle reads updated file. New behavior takes effect immediately without restart (for most files like SOUL.md, MEMORY.md).
6
User restarts agent
Restart button sends POST to /api/restart. Gateway restarts the agent session. Pending work may be lost.
7
Dashboard auto-refreshes
Every 10 seconds, dashboard re-fetches status to keep agent states current. Manual refresh available via "Refresh" button.
